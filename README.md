# Инструкция

## Синтаксис
build/executable [OPTION]... [FILE]...
 
## Описание         
-0, --null: Заменяет переносы строк на NUL в выводе результата 

-a, --all: Дополненяет результата выводом файлов

-A, --apparent-size: Выводит сумму полезных байт в место реального физического пространства. Обычно это значение меньше но может быть и больше из-за "разряжённости" в файлах, внутренней       фрагментации, косвенных блоков и так далее

-B, --block-size=SIZE: Выводит размеры файлов в выбранных еденицах измерения. Допустимые единицы измерения описаны ниже. Пример: '-BM' выводит размер в блоках по 1,048,576 байт

-b, --bytes: Эквивалент '--apparent-size --block-size=1'

-k: эквивалент --block-size=1K

-m: эквивалент --block-size=1M

-c, --total: Добавляет строку с общим итогом

-D, --dereference-args: Позволяет получить для символических ссылок УКАЗАННЫХ ПРИ ЗАПУСКЕ размер конечного файла (за место размера ссылки)

-H: эквивалент --dereference-args (-D)

-L, --dereference: Позволяет получить размер конечного файла для всех символических ссылок
        
-d, --max-depth=N: Выводит результаты только до указанного уровня вложенности. --max-depth=0 аналогично --summarize
        
--files0-from=F: Позволяет Использовать файл F для указания списка файлов и папок через разделитель NUL. При значении F '-' используется стандартный ввод.  

-h, --human-readable: Выводит размеры в человекочитаемом формате (Например: 1K 234M 2G)

--si: эквивалент -h, но использует степени 1000 в место 1024

--inodes: list inode usage information instead of block usage

-l, --count-links: Позволяет учитывать размеры жёстких ссылок более одного раза

-P, --no-dereference: Предотвращает переходы по символическим ссылкам (Поведение по умолчанию)

-S, --separate-dirs: Исключает размер вложенных директорий при расчёте размера директории

-s, --summarize: Отображает только итоговые результаты для каждого из аргументов

-t, --threshold=SIZE: Исключает объекты меньше указанного размера когда SIZE > 0 и объекты больше указанного размера когда SIZE < 0

--time: Выводит время последнего изменения файлов и папок в результате

--time=WORD: Выводит время событий отличных от модификаци на выбор (atime, access, use, ctime or status)
        
--time-style=STYLE: Позволяет задать желаемый формат для вывода времени из атрибута --time. Допустимые опции форматирования описаны ниже, в разделе TIME_STYLE.

-X, --exclude-from=FILE: Исключает из рассчётов файлы и папки попадающие под паттерн(ы) из файла FILE

--exclude=PATTERN: Исключает из рассчётов файлы и папки попадающие под паттерн

-x, --one-file-system: Пропускает директории в файловых систем отличных от фс корневой директории

--help: Выводит текст подсказки

--version: Выводит текущую версию приложения


## Дополнительная информация

### Единицы измерения
Аргумент SIZE - это число с опцинальной единицей измерения (Пример: 10K - это
10*1024).  Eдиницы измерения: K,M,G,T,P,E,Z,Y,R,Q (Степени 1024) или
KB,MB,... (Степени 1000). Бинарный префикс также может быть использован:
KiB=K, MiB=M, и т.д.

### Форматирование даты
Аргумент STYLE из атрибута --time-style может иметь значние full-iso, long-iso, iso, или
+FORMAT.  FORMAT при этом интерпретируется как в ['date(1)'](https://man7.org/linux/man-pages/man1/date.1.html). 
Также для форматирвания по умолчанию можно задать переменную окружения TIME_STYLE.
